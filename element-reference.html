Do's & Don'ts

The most important rules — the ones most likely to be broken accidentally.

✓ Sage text on light bg: Use #2a7068 (dark sage) whenever you want a teal/sage-tone for text on ivory, white, or linen.
❌ Never: color: #C2DED8 on any light background — contrast is 1.6:1, far below WCAG AA.
✓ Header italic: Keep .header-title em { color: #a8cfc8 } — this is the "Research & Resources" color. Don't change it.
❌ No purple text. Purple was the old palette. Replace any purple text with var(--navy) or var(--charcoal).
✓ Nav alignment: Interior pages use justify-content: flex-start on .nav-menu. Home page only: center.
❌ No third typeface. Lora + Inter only. Do not add any additional font family.
✓ header-inner z-index: Always set z-index: 2 on .header-inner so text renders above the helix SVG (z-index: 1).
✓ Max content width: All main content constrained to 860px via max-width: 860px; margin: 0 auto.
❌ Don't use a shared .css file. All styles are inline per-page. This is intentional for GitHub Pages simplicity.
✓ Sand for CTAs: Use var(--amber) (#C6A27E) for all buttons, active states, and call-to-action elements.
